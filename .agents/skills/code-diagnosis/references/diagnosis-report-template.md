# 診斷報告標準格式

> CLI 必須將 `diagnosis_report.md` 按此結構寫入。

```markdown
# 程式碼診斷報告
> 分析時間: {ISO 8601 timestamp}
> 分析範圍: {模組列表}
> 故障症狀: {症狀描述}

## 已讀取檔案
| # | 檔案路徑 | 行數 | 備註 |
|---|----------|------|------|
| 1 | ... | ... | ... |

## 可疑區域（依可能性排序）

### 嫌疑 1: {檔案名:行數範圍}
- **可疑原因**: {詳細說明}
- **相關程式碼**:
  ```
  {可疑程式碼片段，含行號}
  ```
- **建議調查方向**: {給主腦的具體調查建議}

### 嫌疑 2: ...

## 排除區域
| # | 檔案路徑 | 排除理由 |
|---|----------|---------|
| 1 | ... | ... |

## 跨模組關聯分析
- **資料流路徑**: {資料從哪個模組流到哪個模組}
- **依賴鏈**: {涉及的 import/require 關係}
- **共用狀態**: {多個模組共享的狀態或資源}

## 初步結論
{CLI 對故障根因的初步判斷，明確標注「此為初步分析，需主腦複查」}
```
